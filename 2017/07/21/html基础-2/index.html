2、框架

  • 可在同一浏览器窗口中显示不止一个页面，每份HTML文档称为一个框架，且每个框架都独立于其他的框架；
  • 但开发人员必须同时跟踪更多的HTML文档，且很难打印整张页面；

  • 框架结构标签（< frameset >）定义如何将窗口分割为框架，每个frameset定义了一系列行或列；rows/columns 的值规定了每行或每列占据屏幕的面积；

  • 不能将 < body >标签与< frameset >标签同时使用！不过，假如你添加包含一段文本的 < noframes >标签，就必须将这段文字嵌套于< body >< /body >标签内!

  • 导航框架

    注：左侧导航frame包含三个链接，其target为右侧frame的name。导航frame跳转至另一frame内指定节，即一个frame设置了指向另一个文件内指定的节的链接，该节用< a >的name进行标识。

  • 使用iframe作为链接的目标

4、脚本

  • < script >
    • 此元素用于定义客户端脚本，如 JavaScript ;
    • 此元素既可包含脚本语句，也可通过 src 属性指向外部脚本文件；
    • 此元素必需的 type 属性规定脚本的 MIME 类型；
    • JavaScript 最常用于图片操作、表单验证以及内容动态更新；
  • < noscript >
    • 此元素提供无法使用客户端脚本时的替代内容；
  • 实例

注：如果浏览器压根没法识别 < script > 标签，那么 < script > 标签所包含的内容将以文本方式显示在页面上。为了避免这种情况发生，你应该将脚本隐藏在注释标签当中。那些老的浏览器（无法识别 < script > 标签的浏览器）将忽略这些注释，所以不会将标签的内容显示到页面上。而那些新的浏览器将读懂这些脚本并执行它们，即使代码被嵌套在注释标签内。

5、头部

  • < head >元素是所有头部元素的容器，如< title >、< base >、< link >、< meta >、< script >以及< style >，以指示浏览器在何处可以找到样式表，提供元信息，包含脚本等；
  • < base >(为页面上所有链接规定默认地址href或默认目标target)

  • < link >(定义文档与外部资源之间的关系，最常用于连接样式表)

  • < meta >

    • < meta >标签提供关于 HTML 文档的元数据(metadata)(关于数据的信息)；典型的情况是，meta 元素被用于规定页面的描述、关键词、文档的作者、最后修改时间以及其他元数据；< meta > 标签始终位于 head 元素中;
    • 元数据可用于浏览器(如何显示内容或重新加载页面)，搜索引擎(关键词)，或其他 web 服务;
    • < meta >的name和content属性可被搜索引擎用来索引页面；

7、URL

  • scheme://host.domain:port/path/filename
    解释：
    • scheme - 因特网服务类型(http、https、ftp、file等)
    • host - 域主机
    • domain - 因特网域名
    • :port - 主机上的端口号
    • path - 服务器上的路径
    • filename - 文档/资源的名称
