jQuery简介

发表于 2017-07-23

特性与引用

  • HTML元素选取、HTML 元素操作、CSS 操作、HTML 事件函数、JavaScript 特效和动画、HTML DOM 遍历和修改、AJAX、Utilities；

  • 引用：在自己的计算机上存放 jQuery 库然后直接引用，或从 Google 或 Microsoft 加载 CDN jQuery 核心文件；

语法(为HTML元素选取和执行操作编制)

  • $(selector).action()：$定义jQuery、selector查询元素、action执行操作；
  • 所有jQuery函数位于一个document ready函数中，防止文档未完全就绪运行函数导致操作失败；

示例

selector

  • selector允许通过标签名、属性名或内容查询元素；
  • selector允许查询HTML/DOM元素组或单个元素；
  • 元素selector
    • $(“p”) 选取 < p> 元素；
    • $(“p.intro”) 选取所有 class=”intro” 的 < p> 元素；
    • $(“p#demo”) 选取所有 id=”demo” 的 < p> 元素；
    • $(“ul li:first”) 选取每个 < ul> 的第一个 < li> 元素；
  • 属性selector
    • $(“[href]”) 选取所有带有 href 属性的元素；
    • $(“[href=’#’]”) 选取所有带有 href 值等于 “#” 的元素；
    • $(“[href!=’#’]”) 选取所有带有 href 值不等于 “#” 的元素；
    • $(“[href$=’.jpg’]”) 选取所有 href 值以 “.jpg” 结尾的元素；
    • $(“div#intro.head”) 选取id=”intro” 的 < div> 元素中的所有 class=”head” 的元素；
  • CSS selector
    • $(“p”).css(“background-color”,”red”);

2、框架

  • 可在同一浏览器窗口中显示不止一个页面，每份HTML文档称为一个框架，且每个框架都独立于其他的框架；
  • 但开发人员必须同时跟踪更多的HTML文档，且很难打印整张页面；

  • 框架结构标签（< frameset >）定义如何将窗口分割为框架，每个frameset定义了一系列行或列；rows/columns 的值规定了每行或每列占据屏幕的面积；

  • 不能将 < body >标签与< frameset >标签同时使用！不过，假如你添加包含一段文本的 < noframes >标签，就必须将这段文字嵌套于< body >< /body >标签内!

  • 导航框架

    注：左侧导航frame包含三个链接，其target为右侧frame的name。导航frame跳转至另一frame内指定节，即一个frame设置了指向另一个文件内指定的节的链接，该节用< a >的name进行标识。

3、内联框架iframe

  • iframe在网页内显示网页
  • < iframe src=”URL” >< /frame ><!其中URL指向隔离页面的位置>
  • 使用iframe作为链接的目标

4、脚本

  • < script >
    • 此元素用于定义客户端脚本，如 JavaScript ;
    • 此元素既可包含脚本语句，也可通过 src 属性指向外部脚本文件；
    • 此元素必需的 type 属性规定脚本的 MIME 类型；
    • JavaScript 最常用于图片操作、表单验证以及内容动态更新；
  • < noscript >
    • 此元素提供无法使用客户端脚本时的替代内容；
  • 实例

注：如果浏览器压根没法识别 < script > 标签，那么 < script > 标签所包含的内容将以文本方式显示在页面上。为了避免这种情况发生，你应该将脚本隐藏在注释标签当中。那些老的浏览器（无法识别 < script > 标签的浏览器）将忽略这些注释，所以不会将标签的内容显示到页面上。而那些新的浏览器将读懂这些脚本并执行它们，即使代码被嵌套在注释标签内。

5、头部

  • < head >元素是所有头部元素的容器，如< title >、< base >、< link >、< meta >、< script >以及< style >，以指示浏览器在何处可以找到样式表，提供元信息，包含脚本等；
  • < base >(为页面上所有链接规定默认地址href或默认目标target)

  • < link >(定义文档与外部资源之间的关系，最常用于连接样式表)

  • < meta >

    • < meta >标签提供关于 HTML 文档的元数据(metadata)(关于数据的信息)；典型的情况是，meta 元素被用于规定页面的描述、关键词、文档的作者、最后修改时间以及其他元数据；< meta > 标签始终位于 head 元素中;
    • 元数据可用于浏览器(如何显示内容或重新加载页面)，搜索引擎(关键词)，或其他 web 服务;
    • < meta >的name和content属性可被搜索引擎用来索引页面；

7、URL

  • scheme://host.domain:port/path/filename

    • scheme - 因特网服务类型(http、https、ftp、file等)

HTML表单(用于搜集不同类型的用户输入)

  • < form >元素
    • < form >元素用于定义HTML表单;
    • HTML表单包含表单元素,不同类型的 input 元素、复选框、单选按钮、提交按钮等等;
    • < form> … < /form >

  • Action属性

    • 指定某个服务器脚本来处理被提交表单；
    • 如果省略action属性，则默认当前页面；

  • Method属性

    • 规定在提交表单时所用的HTTP方法(GET或POST)

    • GET(默认方法)
      表单被动提交，且没有敏感信息；
      表单数据在地址栏可见；
      适合提交少量数据

    • POST
      适合表单正在更新数据时，或包含敏感信息；
      表单数据在地址栏不可见；
  • Name属性
    • 每个输入字段必须设置一个name属性才能正确提交；

  • < fieldset >组合表单数据(< legend >为< fieldset >定义标题)

    • 实例

桃花源记

发表于 2017-07-20

　　晋太元中，武陵人捕鱼为业。缘溪行，忘路之远近。忽逢桃花林，夹岸数百步，中无杂树，芳草鲜美，落英缤纷。渔人甚异之。复前行，欲穷其林。
　　林尽水源，便得一山，山有小口，仿佛若有光。便舍船，从口入。初极狭，才通人。复行数十步，豁然开朗。土地平旷，屋舍俨然，有良田美池桑竹之属。阡陌交通，鸡犬相闻。其中往来种作，男女衣着，悉如外人。黄发垂髫，并怡然自乐。 见渔人，乃大惊，问所从来。具答之。便要还家，设酒杀鸡作食。村中闻有人，咸来问讯。自云先世避秦时乱，率妻子邑人来此绝境，不复出焉，遂与外人间隔。问今是何世，乃不知有汉，无论魏晋。此人一一为具言所闻，皆叹惋。余人各复延至其家，皆出酒食。停数日，辞去。此中人语云：“不足为外人道也。”
　　既出，得其船，便扶向路，处处志之。及郡下，诣太守，说如此。太守即遣人随其往，寻向所志，遂迷，不复得路。
　　南阳刘子骥，高尚士也，闻之，欣然规往。未果，寻病终，后遂无问津者。
